HyperFurion VK

A voice keyboard for Linux. Press Ctrl+Space, speak, and it types into whatever app has focus.

How it works

A local daemon captures microphone audio, sends it to the speech-to-text provider you configured, and types the transcript into the focused app through a uinput virtual keyboard. It works in terminals, IDEs, browsers — anything that takes keystrokes, on Wayland and X11.

It also does the reverse: select text anywhere and run voice-keyboard tts to hear it read aloud. That works here too — select any sentence on this page and press Ctrl+Alt+T.

Usage

# tap to toggle dictation, or hold to talk
Ctrl+Space

# equivalent CLI
voice-keyboard toggle
voice-keyboard status

# read the selected text aloud
voice-keyboard tts

Configuration

The real config lives at ~/.config/voice-keyboard/config.toml. The copy below is live: edit it and this page re-binds itself. Change the hotkey, switch the mode, tune the hold threshold — then try it.

[hotkey]
# the daemon defaults to control+space; browsers
# already use space for scrolling, so this page
# binds control+alt+v instead
key = "control+alt+v"
# auto = tap to toggle, hold to talk
# other choices: toggle, hold
mode = "auto"
hold_threshold_ms = 280

[tts]
# select text on this page, press ctrl+alt+t
rate = 1.0
pitch = 1.0

Providers

Bring your own API key. Switch providers with one line in the config.

  • Speech-to-text: xAI (streaming), OpenAI, Groq, Deepgram, AssemblyAI
  • Text-to-speech: xAI, OpenAI, ElevenLabs

Notes

  • Linux only. Requires Python 3.11+, systemd user services, and uinput access.
  • Cloud STT/TTS — this is not an offline dictation engine.
  • Recording overlay is GNOME Shell 50 on Wayland; other desktops get notifications.
  • Hotkeys need input-group access; log out and back in once after install.
  • This page runs on your browser's speech engines, the same way the daemon runs on its providers. Nothing else is loaded, stored, or sent.
