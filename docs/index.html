How it works

One daemon. Zero app integration.

HyperFurion VK doesn't hook into editors or browsers. It runs a local systemd user daemon that captures your mic, transcribes it, and types the result through a uinput virtual keyboard — so it works anywhere your cursor blinks.

01

Press Ctrl+Space

Tap to toggle, or hold to talk and release to stop. A near-field overlay on GNOME (notifications elsewhere) shows recording state right where you're typing.

02

Speak

Audio goes to your configured STT provider — streamed live over WebSocket with xAI, or buffered and submitted on stop with the REST providers.

03

Watch it type

The transcript is injected as real keystrokes into whatever app has focus. Terminals, IDEs, chat boxes, Electron apps — anything that takes a keyboard.

Features

Small tool, sharp edges filed off.

🎙️

Global hotkeys, three modes

Tap-to-toggle, hold-to-talk, or auto (both). Reads Linux input events directly — no desktop-environment plugin required.

🔊

Read anything aloud

Select text in any app and run voice-keyboard tts. It reads the primary selection and plays synthesized speech locally.

🔌

Provider-selectable

Five STT and three TTS providers, switchable in one config line. Model IDs are plain config values — new models need no code changes.

⌨️

uinput injection

Types through a kernel virtual keyboard, not clipboard paste hacks. Works on Wayland and X11 alike, in every app.

🖥️

Native desktop feedback

A GNOME Shell overlay anchors recording status near your text caret via AT-SPI. Other desktops get clean notification fallback.

🔒

Honest about privacy

Keys in a 600 config file, socket mode 600, transcript text kept out of default logs. What goes to the cloud is documented, not hidden.

Providers

No surprises

What it is — and isn't.

Good tools state their limits up front.

It is

  • A Linux-first dictation and read-aloud daemon for your whole desktop
  • Desktop-agnostic at the core: uinput + input events, not DE plugins
  • Installable in one script on apt, dnf, and pacman distros
  • MIT-licensed, ~9 focused Python modules, tested

It isn't

  • An offline engine — STT and TTS are provider APIs, key required
  • Available on macOS or Windows
  • A full IME — injection covers printable ASCII, newline, and tab today
  • Magic: hotkeys need input-group access (log out/in once after install)
